# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repository Is

A collection of philosophical essays, fragments, and letters by Buddy Williams, written in Markdown. Topics span AI consciousness, economics, epistemology, and what it means to be human in an age of thinking machines. There is no build system, no tests, no application code — only prose.

## Repository Structure

- `essays/` — Full-length essays (the core work)
- `fragments/` — Shorter pieces, concept sketches, and preserved threads
- `letters/` — Personal pieces written for specific people
- `bridges/` — Companion reader's guides that orient readers before tackling a main essay
- `meta/` — Author info and context
- `assets/` — Images referenced by writings, organized by topic subdirectories
- `style-guide.md` — **Read this before any writing or editing task.** Defines voice, structure, criticism standards, and operating priorities.

## Status System

Each piece has a status in the README table: **Draft** (AI-assisted sketch), **Working** (hand-rewritten with some AI help), **Final** (complete).

## How to Assist with Writing

The `style-guide.md` is the authoritative reference. Key points:

- **Voice**: Clear, serious, exploratory, intellectually alive. Think Thomas Sowell's clarity meets David Deutsch's explanatory ambition. Not academic, not performative.
- **Structure**: Essays follow Abstract → Table of Contents → Introduction → Arguments/Explanations → Conclusion → Appendices.
- **Terminological consistency**: When a term is defined in an essay, use it precisely throughout. Do not swap in near-synonyms that blur distinctions.
- **Criticism**: Actively identify unsupported claims, logical gaps, missing transitions, unstated assumptions, and places where bridging sentences would help. Distinguish preventable confusion from fundamental disagreement.
- **Anticipate objections**: Flag claims stated too strongly or too early for the reader to accept. Suggest bridging clauses, softened framing, or reordering.
- **Priority order when tradeoffs arise**: explanatory clarity > conceptual accuracy > originality > readability > elegance.
- **Avoid**: empty academic language, jargon, pretending certainty, summarizing instead of explaining, softening bold ideas to seem safer.

## Intellectual Context

Rooted in critical rationalism (Popper, Deutsch), systems thinking, and economics as coordination under uncertainty. Influenced by Hayek, Sowell, Sagan, Gödel, Turing, Haidt. The central question: how to design systems that remain humane as intelligence becomes abundant.
